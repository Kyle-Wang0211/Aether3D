## 三、文献与最佳实践

### 3.1 3D 渲染（Splat / Mesh）

| 来源 | 结论 |
|------|------|
| Gaussian Splatting | 3DGS 需球谐系数 + 协方差渲染；PLY 含 positions/scales/rotations |
| RealityKit | `MeshResource`、`ModelEntity` 可直接展示；USDZ 为 AR 首选 |
| Metal | 自定义 splat 渲染需 point/quad 着色器 + 深度排序 |
| Scaniverse/Polycam | 即时 3D 预览为标配，无预览则用户无法感知质量 |

### 3.2 分享与展示

| 来源 | 结论 |
|------|------|
| UIActivityViewController | iOS 标准分享入口 |
| ARQuickLook | 对 USDZ 提供系统级 AR 预览，零代码 |
| Files / 相册 | 导出到本地为基本需求 |

### 3.3 3D 编辑

| 来源 | 结论 |
|------|------|
| QEM (Quadric Error Metrics) | 网格简化业界标准 |
| SceneKit / RealityKit | `SCNNode` 变换、`Transform` 矩阵 |
| 手势 | 双指缩放、单指旋转、拖拽平移 |

## 四、可自研/自建突破方案

### 4.1 P0：3DViewer 真实渲染

**方案 A（Splat）**：  
- 解析 PLY/splat 的 positions、scales、rotations、sh  
- Metal compute/raster 渲染高斯椭球（splat 着色器）  
- 或接入已有 3DGS 渲染库（若可集成）

**方案 B（Mesh）**：  
- 从 TSDF MeshOutput 生成 `RealityKit.MeshResource`  
- 用 `ModelEntity` + `USDZ` 展示，支持 ARQuickLook

**方案 C（最小可行）**：  
- 仅 Mesh：`MeshResource` + `ModelEntity` 在 SceneKit/RealityKit 中显示  
- 或 `ARQuickLook` 预览 USDZ，无需自研渲染

### 4.2 P0：ShareModule

**方案**：  
- 添加分享按钮 → `UIActivityViewController`  
- 支持导出：USDZ（AR）、GLB（通用）、图片（缩略图）  
- 保存到相册、存储到 Files

> **4.3 热力图已删除**：ScanGuidanceRenderPipeline 已实现 displaySnapshot→楔形填充+边框，三角形颜色随覆盖变化，为系统层更优解，无需 GuidanceRenderer 热力图。

### 4.4 P1：3DEditor 基础变换

**方案**：  
- `Transform` 矩阵：scale、rotation、translation  
- 手势绑定：`MagnificationGesture`、`DragGesture`、旋转  
- 可选裁剪：AABB 或平面裁剪

### 4.5 P2：SocialModule

**方案**：  
- 后端：SpaceOwnership、AccessControl、Comment（RFC 设计）  
- 客户端：发布作品、评论、点赞 UI  
- 安全：ABAC、rate limit、审计链

## 七、用户利益总结（大白话）

### 拍摄模块（V2.11 + V2.12，已做）

- **模糊检测**：真的在算每张图清不清晰，糊的会被丢掉，3D 更干净  
- **阈值更严**：更少模糊、过曝、太暗的帧进重建，质量更高  
- **触觉更灵敏**：手机更早振动提示「拍糊了」，有更多时间调整  

### 渲染 / UI / 社交 / 编辑（本报告，待做）

- **3DViewer**：拍完能真的看到 3D 模型，而不是「未实现」占位  
- **ShareModule**：能分享到微信、保存到相册、导出文件  
- **ScanGuidanceRenderPipeline**：已实现三角形楔形填充+边框，颜色随 displaySnapshot 覆盖变化（热力图已删除）  
- **3DEditor**：能缩放、旋转、裁剪模型，微调效果  
- **SocialModule**：能发布作品、看别人的、点赞评论（需后端配合）

## 八、结论

- **拍摄链**：质量检测与阈值已升级，方法论与数值均有突破。  
- **展示链**：渲染、UI、社交、编辑存在大量缺口，影响端到端体验。  
- **建议**：优先落地 3DViewer + ShareModule，形成「拍 → 看 → 分享」闭环；再补 3DEditor、Social。
