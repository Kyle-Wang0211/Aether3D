## 三、修复方式工业对标

### 3.1 已采用的正确实践

1. **显式 padding 命名** (`_pad`, `_pad_end`, `_reserved`) — 符合 Interrupt、GNU C Manual 建议
2. **cache-line 对齐 (64B)** — AetherObservation、AetherAdmissionDecision 已做
3. **去指针化** — AetherSceneSpecOutput 改为 descriptor + caller-provides-buffer，消除 32/64 位差异
4. **typedef enum + sentinel** — 保证存储宽度，符合跨平台 C ABI
5. **byte-level layout 注释** — 每个 struct 有 [offset-size] 标注，便于审计与迁移
6. **static_assert(sizeof)** — 编译期验证，符合 V2.7 登峰造极要求

### 3.2 检索结论（2024-2026）

- **C2.12 / GNU C Manual**：compiler 自动 padding，显式 padding 可 documenting intent
- **Stack Overflow (AArch64/Arm32)**：显式 padding、避免 #pragma packed、field 排序可减 padding
- **P3407 (C++24/26)**：offsetof 用法正在规范化，当前 offsetof 断言为有效手段
- **libffi**：ABI 相关 layout 需显式指定，文档化 layout 为必要
