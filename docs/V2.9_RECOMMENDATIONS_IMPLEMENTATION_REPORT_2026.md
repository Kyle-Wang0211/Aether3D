## 二、PR4Math Fixture 导出器

### 2.1 新增文件

- **Sources/PR4MathFixtureExporter/main.swift**  
  可执行目标，依赖 PR4Math、PR4Softmax、PR4LUT，导出 JSON fixture。

### 2.2 输出格式

- **路径**: `Tests/Fixtures/PR4Math/pr4math_golden_v1.json`
- **结构**:
  - `softmax3`, `softmax5`: input/output/sum (Q16)
  - `exp`: exp0, expNeg1, expNeg2, expNeg10, expNeg32
  - `median`, `mad`: input/output
  - `q16`: add, mul, div (a, b, result, overflow)

### 2.3 使用方式

```bash
swift run PR4MathFixtureExporter
# → Tests/Fixtures/PR4Math/pr4math_golden_v1.json
# 若 aether_cpp/golden 存在，同时复制到该目录
```

### 2.4 与 regen_fixtures 集成

`scripts/regen_fixtures.sh` 已增加 `swift run PR4MathFixtureExporter`，与 FixtureGen 一并执行。

## 三、CI Golden Fixtures 自动验证

### 3.1 新增工作流

- **.github/workflows/golden-fixtures.yml**

### 3.2 触发条件

- **pull_request**: 当以下路径变更时触发  
  - Core/**, Sources/FixtureGen/**, Sources/PR4MathFixtureExporter/**  
  - Sources/PR4Math/**, Sources/PR4Softmax/**, Sources/PR4LUT/**  
  - Tests/Fixtures/**, scripts/regen_fixtures.sh

- **push**: 仅 main 分支，相同路径过滤

### 3.3 执行步骤

1. Checkout  
2. 选择 Xcode 16.2 / 16.1 / default  
3. `swift build`  
4. `bash scripts/regen_fixtures.sh`  
   - 运行 FixtureGen + PR4MathFixtureExporter  
   - 若 `git diff Tests/Fixtures` 非空则失败

## 四、CMake 与 Swift Package 集成构建脚本

### 4.1 新增脚本

- **scripts/build_cpp_and_swift.sh**

### 4.2 用法

```bash
./scripts/build_cpp_and_swift.sh [--cpp-only|--swift-only|--all]
```

| 选项 | 行为 |
|------|------|
| `--cpp-only` | 仅构建 aether_cpp (cmake -B build -S . && cmake --build build) |
| `--swift-only` | 仅构建 Swift Package |
| `--all` (默认) | 先 C++ 后 Swift |

### 4.3 前置条件

- `aether_cpp/` 和 `aether_cpp/CMakeLists.txt` 需已存在（由 CURSOR_MEGA_PROMPT STEP 1 创建）
- 若无 scaffold，脚本会提示并退出

## 五、方法论与数值核查

### 5.1 Laplacian 模糊阈值（200/120）

- **结论**: 维持现状
- **依据**: Swift CoreBlurThresholds SSOT；PyImageSearch 100；3D 重建偏保守；BAGS/DeblurGS 2024–2025 为软路径，Track P 仍以 200 为硬拒收

### 5.2 Niessner 空间哈希

- **结论**: 维持 Y=19349669
- **依据**: Niessner 2013 GitHub；nvblox 2024 沿用同类方案，哈希常数未变

### 5.3 Dempster–Shafer 融合

- **结论**: 维持 DS_EPSILON 1e-9 (Track P)、1e-12 (Track X)
- **依据**: 2024 研究侧重冲突与悖论，本实现已用 sealing 和 epsilon 控制；无明确建议改动当前数值

### 5.4 Q16 定点

- **结论**: 维持现有实现
- **依据**: 与 CrossPlatformDeterminismStrictTests 一致；pr4math_golden_v1.json 已作为 C++ 对照标准
