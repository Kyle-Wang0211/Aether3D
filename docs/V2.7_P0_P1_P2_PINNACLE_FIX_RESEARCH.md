## 二、P0 编译级：struct sizeof 断言

### 2.1 标准依据

- **ISO C**: 6.7.2.1 — 结构体成员对齐与尾部 padding 为 implementation-defined；`sizeof` 包含所有 padding
- **ABI**: ARM64 AAPCS、x86-64 SysV 要求自然对齐；`uint8_t` 后跟 `uint64_t`/`float` 会产生隐式 padding 至 4/8 字节边界

### 2.2 修复原则

1. **移除或放宽 static_assert**：若布局非 ABI 保证，用 `offsetof` 校验关键字段而非 `sizeof`
2. **显式 padding 字段**：在需要可移植 binary layout 时，用 `uint8_t _pad[N]` 显式填充并注释
3. **文档化**：每 struct 注明“假定 little-endian、自然对齐；大端或不同 ABI 需验证”

## 三、P0 语义级：CanonicalFloat::for_hash 带内错误

### 3.1 SEI CERT ERR02-C

> **Avoid in-band error indicators** — Do not return a special value from a function that lies in the function's normal result domain to signal errors.

- `int64_t` 的合法范围包含 −102；无法区分 `AETHER_ERR_NAN_DETECTED` 与真实 canonical 值
- 符合 POSIX/C 惯例：错误码单独返回，数据经 out-parameter

### 3.2 登峰造极方案

```cpp
// 方案 A（推荐）：out-parameter
[[nodiscard]] AetherStatus CanonicalFloat::for_hash(Kind kind, float value, int64_t* out) noexcept;

// 方案 B：tagged union（若需零分配）
struct CanonicalResult {
    AetherStatus status;
    int64_t canon;  // 仅当 status==0 有效
};
[[nodiscard]] CanonicalResult CanonicalFloat::for_hash(Kind kind, float value) noexcept;
```

**主文档采用方案 A**，与 AetherStatus 统一、易检测。

## 五、P0 语义级：ANGLE 范围 [-180, +360]

### 5.1 问题

350° 与 −10° 几何等价，但量化为不同 canonical → 违反「等价输入→相同 hash」

### 5.2 标准与惯例

- **atan2**：主值 (−180°, 180°]（或 (−π, π]）
- **MATLAB/Mapping**：常用 [0, 360) 或 (−180, 180]
- **归一化公式**：`theta_norm = fmod(theta + 180, 360) - 180`（归一化到 (−180, 180]）

### 5.3 登峰造极方案

- **方案 A**：收窄为 `[-180, 180]`，超范围 → `AETHER_ERR_INVALID_RANGE`
- **方案 B**：量化前归一化到 (−180, 180]，再量化；等价角度必得同一 hash

**主文档采用方案 B**，保持向后兼容并满足不变量。

## 六、P0 语义级：ARM64 q16_mul_sat 有符号左移 UB

### 6.1 C++ 标准 [expr.shift]

- 左移负的有符号整数为 **未定义行为**
- `hi << 48` 当 `hi < 0` 时 UB
- 且 ARM 路径缺少 x86 的 MAX_VALUE/MIN_VALUE 饱和检查

### 6.2 登峰造极方案

- 使用无符号中间类型做移位：`(uint64_t)hi << 48` 再组合
- 或直接用 `__int128` 在 ARM64 也可用，与 x86 路径统一
- 必须加饱和检查：`if (result > MAX_VALUE) return MAX_VALUE;` 等

## 七、P0 语义级：VROUNDSS imm8 技术错误

### 7.1 Intel SDM / Felix Cloutier

- imm8 bit 2=1：使用 MXCSR.RC（当前舍入模式）
- imm8 bit 2=0：使用 imm8 bits 1:0
  - 00 = round to nearest (ties-to-even)
  - 01 = round down
  - 10 = round up
  - 11 = toward zero
- **ties-to-even**：imm8 bit2=0, bits1:0=00 → **imm8=0**
- **imm8=8** (0b1000)：bit2=1 → 用 MXCSR，**不保证** ties-to-even

### 7.2 修正

- 主文档中将「mode 8」改为「imm8=0（bits 1:0=00，ties-to-even）」；若用 MXCSR 则需先 `fesetround(FE_TONEAREST)`
