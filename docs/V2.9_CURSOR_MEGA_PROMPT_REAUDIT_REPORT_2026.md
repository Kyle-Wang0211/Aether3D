## 一、新发现并已修复的 Bug

### Bug R1: B.2–B.10 GATE 命令不完整 【已修复】

**问题**：B.1 使用 `cd aether_cpp && cmake -B build -S . && cmake --build build --target math_tests && ctest --test-dir build -R math`，而 B.2–B.10 仅写 `ctest -R merkle` 等。从项目根执行 `ctest -R merkle` 会因缺少 build 目录而失败。

**修复**：B.2–B.10 的 GATE 统一为：

```
cd aether_cpp && cmake -B build -S . && cmake --build build --target <module>_tests && ctest --test-dir build -R <module>
```

并明确 `<module>_tests` 目标名（merkle_tests, time_tests, hash_tests 等）。

---

### Bug R2: BLUR_REJECT 与 Swift SSOT 数值矛盾 【已修复】

**问题**：文档中 `BLUR_REJECT_THRESHOLD = 100.0f`、表「Blur Reject < 100」与 Swift `CoreBlurThresholds.frameRejection = 200` 冲突。Track P 需与 Swift 逐比特一致。

**修复**：
- `BLUR_REJECT_THRESHOLD`: 100 → **200**，并加 SSOT 注释
- 表「Blur Reject < 100」→「< 200」
- 设备归一化注释中的「blur 100」→「blur 200 [frameRejection SSOT]」

---

### Bug R3: C API 行号引用易漂移 【已修复】

**问题**：「copy from MEGA_PROMPT Section 4 C API (lines 1981-2106+struct defs)」依赖行号，文档修改后易失效。

**修复**：改为可搜索引用：

> copy from MEGA_PROMPT "BRIDGE LAYER -- Phase A" + "STRUCT DEFINITIONS" (search: AetherTSDFVolumeRef, AetherObservation, AetherEvidenceSnapshot)

---

### Bug R4: AetherCppBridge 链接说明缺失 【已修复】

**问题**：STEP 3 要求建 AetherCppBridge，但未说明如何将 CMake 生成的 C 库链接进 Swift。

**修复**：在 STEP 3 中补充：
- Package.swift 增加 AetherCppBridge target
- 可选方案：`.systemLibrary` 或 `linkerSettings` + `-L aether_cpp/build`
- 前置条件：CMake 必须生成 `libaether3d_c.a`（或 .dylib）
- STEP 1 中注明：CMakeLists.txt 需包含 `add_library(aether3d_c STATIC ...)`

---

### Bug R5: LAPLACIAN_VAR_REJECT 场景歧义 【已修复】

**问题**：`LAPLACIAN_VAR_REJECT = 50` 未区分 Face/Avatar 与 Scan 场景。50 用于 Face，200 用于 Scan 帧拒收。

**修复**：加注释：

> FACE/AVATAR context only (NOT scan frame rejection). Scan frame rejection: use CoreBlurThresholds.frameRejection = 200 (Track P).

---

### Bug R6: 文档内「blur 100」残留 【已修复】

**问题**：多处仍写「blur 100」「blur_reject x 0.7」等，与 200 不一致。

**修复**：更新为「blur 200」「blur_reject (200) x 0.7」。

## 二、方法论与数值核查

### 2.1 Laplacian 模糊阈值（200 / 120）

| 来源 | 值 | 说明 |
|------|-----|------|
| PyImageSearch | 100 | 基准 |
| Swift CoreBlurThresholds | frameRejection=200, guidanceHaptic=120 | SSOT |
| 文档修复前 | 100 | 与 Swift 不符 |
| 3D 重建实践 | 偏保守 | 保护特征匹配 |

**结论**：200/120 与 Swift、工程实践一致；无需调整。

### 2.2 2024–2025 研究（BAGS、DeblurGS、GeMS）

这些工作采用 blur-aware/agnostic 路径，而非简单拒收。文档已预留软路径（模糊帧可选 BAGS/DeblurGS），Track P 仍以 200 作为硬拒收阈值。
