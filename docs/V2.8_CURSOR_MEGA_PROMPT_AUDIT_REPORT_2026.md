## 一、已修复的 Bug

### Bug 1: FixtureHeader 尺寸描述不一致 【已修复】

| 位置 | 原文 | 问题 |
|------|------|------|
| 多处 | "32-byte FixtureHeader" | `static_assert(sizeof(FixtureHeader) == 36, ...)` 实际为 36 字节 |
| 行 568, 1349, 1354, 13826 | | 4+2+2+4+8+4+2+2+8 = 36 |

**修复**: 将所有 "32-byte" 统一为 "36-byte"。

---

### Bug 2: Phase 0 与自动化工作流脱节 【已修复】

**问题**: Phase 0 标明 “MUST complete before any C++ code”，但原工作流 STEP 0 → STEP 1 之间**没有**生成 Golden Fixtures 的步骤。B.1 要求 "golden fixtures from PR4Math"，自动化会在 B.1 处卡住。

**修复**: 新增 **STEP 0.5: GENERATE GOLDEN FIXTURES**：
1. 运行 `swift run FixtureGen`
2. 创建 `aether_cpp/golden/` 并复制 fixtures
3. 说明 PR4Math golden 来源于 `CrossPlatformDeterminismStrictTests.swift` 的 GoldenValues
4. GATE: `aether_cpp/golden/` 在 B.1 前必须已填充

---

### Bug 3: STEP 1 mkdir 缺少 math/ 和 overflow/ 【已修复】

**问题**: 
- B.1 要求 `aether_cpp/src/math/`，原 mkdir 未包含 `math/`
- B.9 要求 `aether_cpp/src/overflow/`，原 mkdir 未包含 `overflow/`

**修复**: 将  
`src/{tsdf,evidence,merkle,time,quality,scenespec,hash,social}`

`src/{math,tsdf,evidence,merkle,time,quality,scenespec,hash,overflow,social}`

### Bug 4: "Section 2.1" 引用不存在 【已修复】

**修复**: 改为 “copy AetherStatus enum from ‘ERROR STRATEGY -- AetherStatus Return Codes’ section (search: typedef enum { AETHER_OK)”。

---

### Bug 5: 项目路径歧义 【已修复】
