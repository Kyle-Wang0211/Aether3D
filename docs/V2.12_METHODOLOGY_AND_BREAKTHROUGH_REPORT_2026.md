## 二、文献与行业最佳实践

### 2.1 模糊检测（Blur Detection）

| 来源 | 结论 |
|------|------|
| PyImageSearch, TheAILearner | Laplacian 方差为实时场景首选，无外部依赖 |
| renor.it | 将感知质量转为可量化分数，适合客户端预检 |
| Apple Accelerate | vImage 卷积实现 Laplacian，硬件加速 |
| BAGS (ECCV 2024) | 模糊感知 3DGS：多尺度核建模、质量掩码；可作为**软路径**（模糊帧不拒收，走 BAGS） |
| DeblurGS (2024) | 运动模糊 6-DOF 估计、高斯致密退火；视频采集场景 |

**自研路径**: 用 vImage 实现 Laplacian 3×3 卷积，计算方差。Laplacian kernel: `[[0,1,0],[1,-4,1],[0,1,0]]`。

### 2.2 帧重叠估计（Overlap Estimation）

| 来源 | 结论 |
|------|------|
| WWDC 2024 Object Capture | Area 模式下强调图像重叠，相机位姿可视化 |
| OmniGlue (CVPR 2024) | 可学习特征匹配，跨域泛化 |
| 局部单应性 + RANSAC | 平面运动近似，过滤误匹配 |
| Visual Overlap Prediction | ViT 预测重叠分数，无需昂贵特征检测 |

**自研路径**: Vision `VNGenerateImageFeaturePrintRequest` 或 `VNDetectContoursRequest` + 特征匹配；或 CoreML 视觉重叠预测模型（若可集成）。

### 2.3 作业调度

| 来源 | 结论 |
|------|------|
| Apple QoS | User-interactive > User-initiated > Utility > Background |
| EDF (Earliest Deadline First) | 实时系统最优，按 deadline 排序 |
| iOS 26 后台 | 有限后台时间、系统调度 |

**升级路径**: 将 Job 优先级映射到 QoS；可选 EDF 排序（对 deadline 敏感任务）。

### 2.4 安全与时间锚定

| 来源 | 结论 |
|------|------|
| RFC 3161 TSA | 时间戳协议，需 ASN.1 编解码 |
| Roughtime | 云flare 公钥、UDP、Ed25519 签名 |
| OpenTimestamps | 日历服务器、指数退避轮询 |
| swift-crypto | SHA-3-256 规划中，DualDigest 暂用 SHA-256 |

## 三、可自研/自建突破方案

### 3.1 P0：BlurDetector 真实 Laplacian 实现

**方案**: 新增 `LaplacianVarianceComputer`，使用 Accelerate vImage 卷积。

- **输入**: 灰度缓冲 `(UnsafeRawPointer, width, height, rowBytes)`
- **步骤**: 1) Laplacian 3×3 卷积 2) 计算结果方差
- **输出**: `Double`（方差值）
- **调用链**: App 层从 `ARFrame.capturedImage` 取 CVPixelBuffer → 转灰度 → 调用 Computer → 注入 BlurDetector

**实施**: 见 `Core/Quality/Metrics/LaplacianVarianceComputer.swift`（新建）。

### 3.2 P0：FrameData 支持像素维度

**方案**: `FrameData` 增加可选 `width: Int?`, `height: Int?`。

- 当两者均存在且 `imageData.count >= width * height` 时，BlurDetector 可尝试从 imageData 计算 Laplacian。
- 向后兼容：nil 时沿用现有 placeholder 行为。

### 3.3 P1：OverlapEstimator 特征匹配路径

**方案**: 集成 Vision 特征请求，估算重叠比。

- `VNGenerateImageFeaturePrintRequest` 提取特征
- 或 `VNHomographicImageRegistrationRequest`（若可用）
- 备选：两帧各取 grid 采样，用简单 NCC/SSD 近似重叠（低精度但可落地）

### 3.4 P1：MotionAnalyzer IMU 融合

**方案**: `#if canImport(CoreMotion)` 下集成 CMMotionManager。

- 角速度 + 帧差 → 运动分数
- 高通滤波检测 >5Hz 抖动

### 3.5 P2：BAGS/DeblurGS 软路径（Track X）

**方案**: 模糊帧不硬拒收，走模糊感知 3DGS 管线。

- 需后端/离线支持 BAGS 或 DeblurGS。
- 客户端保持严格拒收（frameRejection=200），软路径为未来扩展。

### 3.6 P2：Job 优先级与 QoS 映射

**方案**: `JobStateMachine` 或调度入口按 `job.priority` 设置 `DispatchQoS`。

- 高优先级 → `.userInitiated`，低 → `.utility` 或 `.background`。

### 4.2 App 层接入说明

`ScanViewModel.estimateBlurVariance(from:)` 当前返回固定 200。若要启用真实 Laplacian：

1. 从 `ARFrame.capturedImage` 取 CVPixelBuffer
2. 转灰度（或取 Y 平面）
3. 构造 `FrameData(index: 0, timestamp: ..., imageData: grayData, width: w, height: h)`
4. 调用 `QualityAnalyzer.analyzeFrame(frame)` 或直接 `BlurDetector().detect(frame:)` 获取 `BlurResult.variance`
5. 将 variance 用于触觉/UI 反馈

亦可直接调用 `LaplacianVarianceComputer.compute(bytes:width:height:rowBytes:)` 当已有灰度缓冲时。

## 六、结论

- **方法论缺口**: 质量检测链（Blur/Tenengrad/Texture/Overlap/Motion 等）大量 placeholder，导致质量门、引导、拒收逻辑基于假数据。
- **突破路径**: 自研 Laplacian（vImage）、Vision 特征、CoreMotion 融合；可选 BAGS 软路径、Job QoS 映射。
- **优先级**: P0 聚焦 Blur + FrameData；P1 为 Overlap、Motion；P2 为安全、时间锚定、软路径。

方法论升级与数值升级并行推进，方可达到「极致效果、极致性能、极致安全」目标。
