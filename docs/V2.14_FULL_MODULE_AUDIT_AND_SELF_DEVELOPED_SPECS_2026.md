## 三、自研算法实施规格（可照此完美复现）

### 3.1 TenengradDetector

**文件**：`Core/Quality/Metrics/TenengradDetector.swift`

**步骤**：
1. 新建 `TenengradVarianceComputer`（类同 LaplacianVarianceComputer）
2. Sobel Gx kernel：`[[-1,0,1],[-2,0,2],[-1,0,1]]`（row-major Int16）
3. Sobel Gy kernel：`[[-1,-2,-1],[0,0,0],[1,2,1]]`
4. vImage `vImageConvolve_Planar8` 两次，得 GxBuf、GyBuf
5. 逐像素 Gx²+Gy²；vDSP `vDSP_sve` 求和；除以 N 得 T
6. TenengradDetector.detectFull/Degraded 调用 Computer，传入 frame（需 width/height）
7. 无 width/height 时沿用常量占位

**验收**：清晰图 T>50，模糊图 T<30；单元测试 TenengradVarianceComputerTests。

### 3.2 OverlapEstimator

**文件**：`Core/Quality/OverlapEstimator.swift`

**方案 A**：Vision `VNTranslationalImageRegistrationRequest`（若有）→ 得 transform → 推算重叠区域
**方案 B**：`VNGenerateImageFeaturePrintRequest` 两帧 → `distance`；标定 distance→overlap 曲线（需采集数据）
**方案 C**：16×16 网格，每块 32×32 像素，两帧对应块做 NCC；overlap ≈ 高相关块数/总块数

**验收**：返回非常量；forward 通常 > side；与人工标注重叠相关性 >0.5。

### 3.3 MotionAnalyzer

**文件**：`Core/Quality/Metrics/MotionAnalyzer.swift`

**说明**：三角形颜色不变时用户自然知道要慢下来，无需「动太快/手抖会提示」。MotionAnalyzer 主要用于内部质量门或数据采集。

**步骤**（若需真实计算）：
1. `#if canImport(CoreMotion)` 集成 `CMMotionManager`
2. `startGyroUpdates()` → `gyroData.rotationRate`；积分→角位移 proxy
3. 若可获取帧间 `cameraTransform`，取 translation 差分→线速度
4. `calculateMotionScore`：与真实运动相关，返回非常量

**验收**：返回非常量；内部质量门可用。

### 3.4 InformationGainCalculator

**文件**：`Core/Quality/Admission/InformationGainCalculator.swift`

**computeInfoGain**：
- patch 对应的网格单元在 existingCoverage 中未覆盖比例
- infoGain = 1 - (已覆盖面积 / patch 面积)；边界 [0,1]

**computeNovelty**：
- patch 中心与 existingPatches 中心最近距离；距离越大 novelty 越高
- 或：外观直方图差异；归一化到 [0,1]

**验收**：完全覆盖区新 patch→infoGain≈0；全新区域→infoGain≈1；输出严格 [0,1]。

> **热力图已删除**：系统层 ScanGuidanceRenderPipeline 已实现 displaySnapshot→楔形填充+边框，三角形颜色随覆盖变化，为更优解。
