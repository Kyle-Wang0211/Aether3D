# V2.15 核心层与系统层融合审计与迁移规格

**日期**: 2026-02-12  
**背景**: 热力图与楔形填充分离、displaySnapshot 重复实现等现象暴露核心层(Core)与系统层(App)连接不紧密  
**目标**: 新核心层完美迁移、替换老实现、完美连接系统层；该删的删、该加的加  
**主文档**: 完整规格已融入 `CURSOR_MEGA_PROMPT_V2.md` §6.4.10；本文为可执行参考与审计细节

**设计决策：删除模糊触觉** — 不替用户判断糊不糊；模糊帧静默丢弃，用户从覆盖不增长自然感知。BlurDetector 仅帧级拒收，不用于扫描引导触觉。

## 二、缺口清单（该加的加）

### 2.1 displaySnapshot：用 PatchDisplayMap 替代 dict

**问题**：ScanViewModel 自建 `displaySnapshot`，未用 Core `PatchDisplayMap`。

**影响**：
- PR7 设计：PatchDisplayMap 提供 EMA 平滑、locked 加速、单调性保证
- 现状：线性累加，无 EMA，无与 Evidence 链路的统一可能

**修复规格**：

1. **ScanViewModel 变更**（`App/Scan/ScanViewModel.swift`）：
   - 删除：`private var displaySnapshot: [String: Double]`、`previousDisplay`
   - 新增：`private let patchDisplayMap = PatchDisplayMap()`
   - `updateDisplaySnapshot()` 改为：
     ```swift
     private func updateDisplaySnapshot() {
         let nowMs = Int64(Date().timeIntervalSince1970 * 1000)
         for triangle in meshTriangles {
             let current = patchDisplayMap.display(for: triangle.patchId)
             // 补偿 EMA：target 需略大于线性增量，使约 500 帧到 1.0
             let target = min(current + ScanGuidanceConstants.scanDisplayIncrementPerFrame, 1.0)
             _ = patchDisplayMap.update(
                 patchId: triangle.patchId,
                 target: target,
                 timestampMs: nowMs,
                 isLocked: false
             )
         }
     }
     ```
   - 获取 displaySnapshot 供 Pipeline：
     ```swift
     let displaySnapshot = Dictionary(
         uniqueKeysWithValues: patchDisplayMap.snapshotSorted().map { ($0.patchId, $0.display) }
     )
     ```
   - `resetSubsystems()` 中：`patchDisplayMap.reset()` 替代 `displaySnapshot.removeAll()`
   - `previousDisplay`：从 `patchDisplayMap.snapshotSorted()` 构建，用于 flip 阈值检测

2. **常量新增**（`Core/Constants/ScanGuidanceConstants.swift`）：
   - `scanDisplayIncrementPerFrame: Double = 0.01`（与 EvidenceConstants.patchDisplayAlpha=0.2 配合，约 500 帧到 1.0）
   - 或复用/引用 EvidenceConstants

3. **验收**：
   - 楔形填充颜色变化与之前视觉一致（或更平滑）
   - `PatchDisplayMapTests`、`MonotonicityStressTests` 仍通过
   - 无新 import 违反 Core 边界（App 依赖 Aether3DCore）

### 2.2 删除模糊触觉 + BlurDetector 仅帧级拒收

**设计决策（§6.4.10.4）**：不替用户判断「糊不糊」；不因模糊震动。与「动太快/手抖不提示」一致 — 用户从覆盖不增长、三角形颜色不变自然感知。

**实施**：
- **删除**：ScanViewModel 中 `blurVariance < hapticBlurThreshold` 时 `hapticEngine.fire(pattern: .blurDetected, ...)` 的调用
- **BlurDetector**：仅用于**帧级拒收**（SfM/重建管线）：低于 `FrameQualityConstants.blurThresholdLaplacian` 的帧不送入 SfM；不在扫描引导流程触发触觉
- **estimateBlurVariance**：若 ScanViewModel 仅用于引导、无 SfM 管线，可保留常量返回或移除；若下游有帧级拒收需求，再接入 BlurDetector 供拒收逻辑使用

**验收**：扫描时无模糊震动；糊帧在 SfM 入口被丢弃（若有该管线）。

### 2.3 新增 Core 桥接 API（可选）

为减少系统层对 Evidence 细节的依赖，可在 Core 增加：

- `PatchDisplayMap.displaySnapshot() -> [String: Double]`：返回 `Dictionary(uniqueKeysWithValues: snapshotSorted().map { ($0.patchId, $0.display) })`

此项为便利方法，非必须；系统层可直接构建。

## 七、用户好处（大白话）

- **PatchDisplayMap 替换**：显示更平滑，未来可与 Evidence 统一，少重复逻辑
- **删除模糊触觉**：系统不替用户判断，用户从覆盖不增长自然感知，体验更自然
- **BlurDetector**：仅帧级拒收（糊帧不送入 SfM），不用于引导触觉
- **整体**：核心层算法真正被系统层使用，该删的删、该加的加，架构更清晰
