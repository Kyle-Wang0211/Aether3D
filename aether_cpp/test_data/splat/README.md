# Phase 6 cross-validation test scenes

Two scenes per the locked Phase 6 v2 plan (decision **B**):

## 1. `synthetic_smoke.ply` — dev-iteration scene

100 deterministic Gaussians on a 5×5×4 grid, single SH degree,
opacity = 0.8, colored by grid position. Generated by
`scripts/gen_synthetic_smoke_ply.py`. Loads in < 100 ms,
runs at full speed even on a laptop GPU. Use for inner-loop
debug + smoke test.

**Committed to repo** (small, ~6 KB).

Re-generate from scratch:
```bash
python3 scripts/gen_synthetic_smoke_ply.py \
    > aether_cpp/test_data/splat/synthetic_smoke.ply
```

## 2. `garden_50k.ply` and `garden_500k.ply` — Mip-NeRF 360 garden

The 3DGS-paper-academic-gold-standard scene Mobile-GS benchmarks
against. Same scene = directly comparable perf numbers vs the
published 60 FPS / 116 FPS bars.

**NOT committed** (hundreds of MB; too large for git LFS-free repo).

### Download

The Mip-NeRF 360 dataset is hosted by Jonathan T. Barron / Google:
- Dataset page: https://jonbarron.info/mipnerf360/
- 3DGS-trained checkpoint hosting varies; common sources:
  - INRIA's 3DGS reference: https://repo-sam.inria.fr/fungraph/3d-gaussian-splatting/datasets/
  - gsplat repo training outputs (recommended; reproducible)
  - Polycam / Luma sample exports (look for "garden")

### Download instructions

```bash
# From repo root, with a Mip-NeRF 360 garden 3DGS checkpoint URL
mkdir -p aether_cpp/test_data/splat
cd aether_cpp/test_data/splat

# Example (URL is illustrative — replace with actual checkpoint URL):
curl -L -o garden_500k.ply \
    "https://example.com/mipnerf360/garden/point_cloud/iteration_30000/point_cloud.ply"

# Optionally subsample to 50k for the headline-perf benchmark:
python3 ../../../scripts/subsample_ply.py \
    --input garden_500k.ply \
    --output garden_50k.ply \
    --target-count 50000 \
    --strategy importance  # weight by opacity * scale norm
```

### Expected file sizes

- `garden_50k.ply`: ~5-10 MB
- `garden_500k.ply`: ~50-100 MB
